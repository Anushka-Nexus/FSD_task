React: Library vs Framework
React as a Library

React (developed by Facebook/Meta) is officially a JavaScript library for building user interfaces (UIs).

It focuses only on the view layer â€” meaning, how things appear on the screen.

When you use React alone, you handle routing, state management, and APIs yourself or with other tools.

Example:

  return <h1>Hello React!</h1>;

React as a Framework (with Next.js, React Router, etc.)

When you combine React with routing (React Router), data fetching, and server-side rendering (Next.js), it becomes more like a framework.

Frameworks built on React (like Next.js) add:

Routing

Server-side rendering (SSR)

Static site generation (SSG)

API routes

Better performance and SEO

So React by itself = library,
React + ecosystem (Next.js, etc.) = framework-like environment.

ğŸŒ 7. SPA (Single Page Application)

React apps are Single Page Applications (SPAs).

There is only one HTML file (usually index.html).

React updates the DOM dynamically using a Virtual DOM.

Navigation feels like switching pages, but no full page reload happens.

Advantages:

Faster navigation

Better user experience

Reusable UI components

Disadvantages:

SEO can be tricky (solved by Next.js)

Initial load can be heavier
